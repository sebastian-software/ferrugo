# Mesh Shading Tessellation Report

Date: 2026-06-25.
Milestone: 0108.

## Summary

Milestone 0108 adds the first Rust-native mesh shading slice without widening
the PDFium runtime surface. The native renderer now resolves stream-backed
shading resources and decodes a bounded Type 4 free-form Gouraud triangle mesh
subset:

- `/DeviceGray` and `/DeviceRGB` color spaces.
- 8-bit coordinates and 8-bit color components.
- 2- or 8-bit mesh flags.
- Explicit flag-0 triangle records.

Continuation flags, Type 5/6/7 mesh shadings, and non-8-bit coordinate/component
encodings remain typed unsupported boundaries. The existing
`mesh-shading-unsupported.pdf` fixture is intentionally still a fallback case.

## Budgets

Mesh decoding is bounded before rasterization:

- `DisplayListOptions::max_mesh_shading_bytes`: default 1 MiB decoded stream
  data.
- `DisplayListOptions::max_mesh_shading_triangles`: default 8192 decoded
  triangles.

Both limits map through the native backend to `renderer.memory-budget`. The
triangle rasterizer clips to device bounds and shades only decoded triangles, so
work stays tied to output pixels and bounded triangle count.

## Fixtures

Added:

- `fixtures/generated/type4-mesh-shading.pdf`

The fixture is generated by `scripts/generate_fixtures.py` and records one Type 4
triangle with red, green, and blue Gouraud vertex colors. It is registered as a
report-family corpus fixture.
