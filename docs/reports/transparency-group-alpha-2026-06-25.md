# Transparency Group Alpha Report

Date: 2026-06-25.
Milestone: 0109.

## Summary

Milestone 0109 improves transparency group compositing in the Rust-native
renderer while keeping the implementation PDFium-guided. The group path now:

- Preserves alpha when painting onto transparent intermediate rasters.
- Applies caller graphics-state alpha and blend mode when the group is painted
  back to the page.
- Renders the full nested display list into the group surface instead of
  path-only content.
- Keeps the intermediate surface clipped to the transformed group bounds and
  bounded by `PathRasterOptions::max_transparency_group_pixels`.

This directly fixes the prior intermediate-surface behavior where alpha
operations became opaque too early. On normal opaque page backgrounds, existing
path and image alpha output remains unchanged.

## Knockout Boundary

The new fixture uses `/Group << /S /Transparency /I true /K true /CS /DeviceRGB
>>` plus inherited ExtGState alpha inside the Form XObject. The local PDFium
oracle renders the overlap as normal semi-transparent group composition. Native
follows that oracle behavior for this milestone rather than introducing a
hard-knockout interpretation that would create visible divergence.

This keeps `/K true` covered by fixture evidence while leaving deeper
conformance work for a future transparency conformance milestone.
