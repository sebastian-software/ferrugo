# Corpus Taxonomy

Status: accepted for Phase 9 sampling.
Date: 2026-06-24.

The committed corpus stays generated and license-safe. It is grouped by document
family so native renderer progress can be reported against realistic document
shapes instead of only isolated PDF features.

## Families

| Family | Meaning | Typical blockers |
| --- | --- | --- |
| `office-export` | Word processor and spreadsheet-style exports with text, fonts, encodings, and simple layout. | fonts, CMaps, text positioning |
| `browser-print` | Browser print-to-PDF output with page geometry, vector paths, clipping, and inline images. | clipping fidelity, inline images, color spaces |
| `scan` | Scan-like image-dominant pages and image codec/color-space coverage. | image filters, predictors, color conversion, memory budgets |
| `report` | Report/dashboard-style pages with charts, strokes, gradients, patterns, and transparency. | shadings, patterns, blend modes, stroke fidelity |
| `form` | Static form, widget, and AcroForm appearance coverage. | dynamic appearances, XFA, script-backed forms |
| `presentation` | Slide-like or layered documents with Form XObjects and optional content. | optional content policy, nested forms, transparency groups |
| `mixed-layout` | Documents that combine structural features such as annotations, incremental updates, encryption markers, and recovery cases. | annotations, xref recovery, encryption policy |

## Committed Manifest

`fixtures/corpus-manifest.tsv` is the source of truth for committed fixture
family assignment. Each row records:

- relative PDF path
- corpus family
- reproducible source
- fixture license
- expected page count
- coarse feature tags
- source notes

All current committed rows are generated by `scripts/generate_fixtures.py` and
licensed as `MIT OR Apache-2.0`, matching this repository.

## Real-World-Style Manifest

`fixtures/real-world-style-manifest.tsv` is a privacy-safe bridge between the
generated fixture set and production-shaped document categories. It reuses
small generated PDFs but classifies them as invoice, statement, scanned packet,
form, browser export, office export, report, presentation, secure document, and
malformed recovery samples.

The file intentionally keeps the same seven-column TSV shape as
`fixtures/corpus-manifest.tsv`, so existing manifest parsing, fallback summary,
metadata extraction, and benchmark commands work without a second parser.
Expected backend behavior is encoded in feature tags such as
`expected:native`, `expected:native-unsupported`, and
`expected:error-encrypted`.

Use this manifest for retirement planning only as a first synthetic-realistic
proxy. It does not replace real local corpus ingestion.

## Cross-Producer Fusion Manifest

`fixtures/cross-producer-fusion-manifest.tsv` groups equivalent workflows across
multiple producer-style reductions. It keeps the standard seven-column manifest
shape and adds `producer:*` plus `workflow:*` feature tags so native gates and
producer reports can answer two questions at once:

- whether a typical workflow remains native-only across several producers;
- whether a producer-specific boundary routes to a typed native cause.

The companion `fixtures/cross-producer-fusion-matrix.tsv` records workflow,
producer, feature pressure, current status, owner route, and minimization notes.
All rows must remain generated, privacy-safe, and reproducible.

`benchmark-batch-native --pages-per-input` and repetitions to exercise hundreds
of ordered page jobs without committing very large PDFs.

Use this manifest when a change may affect page ordering, cancellation,
high-count scheduler behavior, or memory reporting:

```sh
cargo run -p ferrugo --no-default-features -- benchmark-batch-native \
  fixtures/generated \
  --manifest fixtures/high-page-count-batch-manifest.tsv \
  --include-family long-document \
  --include-family book \
  --include-family email-thread \
  --include-family repeated-resources \
  --include-family report-statement \
  --repetitions 10 \
  --pages-per-input 12 \
  --max-workers 4 \
  --max-in-flight-pixels 102400 \
  --max-edge 160 \
  --max-p95-ms 1000 \
  --max-errors 0 \
  --fail-on-budget
```

## Private Local Corpora

Private or third-party PDFs stay outside Git under `fixtures/local-corpus/`.
Describe them in `fixtures/local-corpus/metadata.toml` using aggregate
`[[sample]]` entries copied from `fixtures/local-corpus.example.toml`. Do not
include filenames, customer names, file hashes, extracted text, screenshots, or
rendered outputs when reporting private corpus results.

Recommended private sampling fields:

- `category`: one of the production-shaped categories from the corpus intake
  policy
- `privacy`: one of `public-redistributable`, `public-reference-only`,
  `private`, or `synthetic-reduced`
- `permission`: redistribution or local-review permission classification
- `redaction_state`: whether the sample is anonymized, not shareable, or
  reduced to a committed fixture
- `count`: number of PDFs in that category
- `page_count_range`: coarse range, for example `1`, `2-10`, `11-50`, `50+`
- `features`: coarse tags such as `fonts`, `forms`, `scanned`, `annotations`
- `source_note`: anonymized provenance such as `internal invoice export`
- `synthetic_replacement`: a `fixtures/generated/*.pdf` replacement or
  `none-yet`
- `status`: `candidate`, `reviewed`, `blocked`, or `reduced`

Use aggregated pass rates by family and fallback category when sharing local
results.

Validate the local metadata shape without exposing document-level details:

```sh
cargo run -p ferrugo --no-default-features -- validate-local-corpus \
  fixtures/local-corpus/metadata.toml --allow-missing
```

## Local Reporting

Use the manifest with the fallback summary command to report native pass rates
by family:

```sh
cargo run -p ferrugo -- summarize-fallbacks fixtures/generated \
  --manifest fixtures/corpus-manifest.tsv \
  --max-edge 120 \
  --output target/ferrugo-thumbnails/fallback-summary.json
```

Use metadata extraction to capture page count, page size, source/license notes,
and coarse feature tags for committed fixtures:

```sh
cargo run -p ferrugo -- extract-corpus-metadata fixtures/generated \
  --manifest fixtures/corpus-manifest.tsv \
  --output target/ferrugo-thumbnails/corpus-metadata.json
```
