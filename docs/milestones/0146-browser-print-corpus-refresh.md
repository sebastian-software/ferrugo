Status: todo
Phase: 27
Size: medium
Depends on: 0145

## Goal

Expand browser print-to-PDF coverage for common web documents such as receipts,
articles, dashboards, invoices, documentation pages, and forms.

## Scope

- Add fixtures generated by current Chromium, Firefox, and WebKit-style print
  pipelines where practical.
- Cover CSS backgrounds, web fonts, links, tables, clipped content, and images.
- Track browser generator differences in fixture metadata.
- Classify native renderer gaps by feature and browser family.

## Non-Goals

- Build an HTML renderer.
- Depend on live web pages for deterministic tests.
- Preserve interactive browser behavior.

## Deliverables

- Browser-print fixture refresh.
- Family-level pass/fail report.
- Renderer gap backlog for common web-print features.

## Acceptance Criteria

- Browser-print fixtures cover text-heavy, mixed-media, and form-like pages.
- Native rendering is deterministic across repeated runs.
- Unsupported cases have typed fallback reasons.

## Validation

- Run corpus manifest validation.
- Run browser-print visual comparison.
- Run repeated native renders for determinism.
- Run native-only `cargo test`.
