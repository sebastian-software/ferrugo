# 0004: License Files And Attribution Policy

Status: todo
Phase: 0
Size: small
Depends on: 0002

## Goal

Turn the MIT/Apache-2.0 license intent into concrete repository files and an
attribution policy for PDFium-guided work.

## Scope

- Add license files for the chosen dual license.
- Add a short attribution policy for PDFium-informed modules.
- Document which external projects may be used as behavior references only.

## Non-Goals

- Perform legal review.
- Import PDFium source code.
- Decide npm package licensing.

## Deliverables

- Repository license files.
- A concise attribution section in the documentation.
- A rule that AGPL code must not enter the core library.

## Acceptance Criteria

- The repository has explicit license files.
- The docs distinguish behavioral reference, architecture inspiration, and code
  porting.
- Direct code porting requires attribution and review.

## Validation

- Confirm license files exist.
- Search docs for contradictory license statements.

Empty until done.
